### Описание проекта

Мы работаем дата-сайентистом в компании Booking. Одна из проблем компании — это нечестные отели, которые акручивают себе рейтинг. Одним из способов обнаружения таких отелей является построение модели, которая предсказывает рейтинг отеля. Если предсказания модели сильно отличаются от фактического результата, то, возможно, отель ведёт себя нечестно, и его стоит проверить. Нам поставлена задача создать такую модель.

### Метрика качества

Результаты оцениваются по метрике MAPE (Mean Absolute Percentage Error) — средняя абсолютная процентная ошибка. Чем ниже значение этой метрики, тем точнее предсказания модели. Файл представления Для каждого **id** отеля в наборе тестовых данных должны предсказать рейтинг отеля для **reviewer_score** переменной. Файл должен содержать заголовок и иметь следующий формат: ``` reviewer_score,id 1,1 ```

### Какой кейс решаем?

Нам необходимо разработать модель на основе алгоритма **Random Forest**, которая будет предсказывать рейтинг отеля (`reviewer_score`) с максимальной точностью. Для этого нужно:

* Очистить данные от пропусков и дублей.
* Извлечь максимум полезной информации из текстов отзывов и метаданных.
* Отобрать наиболее значимые признаки, исключив мультиколлинеарность.

### Краткая информация о данных

В работе используется датасет, содержащий сведения о 515 000 отзывов на отели Европы. Оригинальный набор данных включает:

* Текстовые отзывы (позитивные и негативные).
* Географические координаты и адреса отелей.
* Временные метки и теги рецензентов.
* Итоговые оценки, которые послужили целевой переменной.

### Этапы работы над проектом

1. **Первичный анализ и EDA:** Исследование структуры данных, визуализация распределения оценок и выявление ключевых проблем датасета (пропуски в координатах, неинформативные признаки).
2. **Преобразование и очистка данных:**
    * Восстановление пропущенных координат отелей через API `Geopy` в три этапа.
    * Очистка данных от ошибок типизации (борьба с `pd.NA`) и приведение признаков к числовым форматам.
    * Генерация временных признаков на основе даты отзыва.
3. **Feature Engineering и NLP-анализ:**
    * **Анализ тональности:** Использование `SentimentIntensityAnalyzer` для оценки эмоциональной окраски отзывов.
    * **Векторизация текстов:** Применение метода **TF-IDF** для извлечения 150 наиболее значимых слов-маркеров.
    * **Гео-признаки:** Расчет расстояния до центра городов и кластеризация отелей по районам (K-Means).
4. **Визуализация и отбор признаков:**
    * Построение графиков в стиле **Dark Mode** для анализа значимости признаков (ANOVA F-test, Хи-квадрат).
    * Устранение мультиколлинеарности (удаление признаков с корреляцией > 0.7).
5. **Обучение модели и оценка:**
    * Обучение модели **Random Forest Regressor** на финальном наборе из 182 признаков.
    * Достижение целевого результата **MAPE 11.98%**.

## Итоговые результаты

В ходе работы над проектом удалось значительно повысить качество предсказания за счет глубокого проектирования признаков (Feature Engineering):

* **Финальная метрика (MAPE):** 11.98%.
* **Прогресс:** Точность модели была улучшена с базового уровня **12.04%** до **11.98%**. Ключевой прирост дало внедрение методов NLP (TF-IDF).
* **Масштаб:** Итоговая модель учитывает **182 признака**, включая географические, временные, статистические и семантические данные.
* **Инсайт:** Анализ важности признаков показал, что слова-маркеры из отзывов (например, *dirty, excellent, staff*) имеют сопоставимое влияние на рейтинг отеля с такими важными факторами, как средняя оценка отеля или его местоположение.

### Использованные инструменты и библиотеки

## 🛠 Инструменты и библиотеки

В проекте использован стандартный стек технологий Data Science для обработки данных и машинного обучения:

* **Анализ данных:** `Pandas`, `Numpy` — базовая обработка и манипуляция данными.
* **Визуализация:** `Matplotlib`, `Seaborn` — построение графиков в стиле Dark Mode и анализ корреляций.
* **NLP (Текстовый анализ):** `NLTK`, `TfidfVectorizer` — извлечение тональности (Sentiment Analysis) и векторизация слов.
* **Машинное обучение:** `Scikit-learn` — реализация модели RandomForest, кластеризация K-Means и расчет статистических метрик (ANOVA, Хи-квадрат).
* **Гео-данные:** `Geopy` — работа с географическими координатами и расчет расстояний до центров городов.
